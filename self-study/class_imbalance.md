# Дисбаланс классов

Дисбаланс классов в машинном обучении &ndash; это неравномерное распределение целевых меток в выборке: один (или несколько) класс (классов) значительно преобладает над другим (другими). Это приводит к тому, что модели склонны игнорировать редкий (миноритарный) класс, выдавая высокую точность (*accuracy*) за счет предсказания только доминирующего класса.

Существует несколько подходов для решения данной проблемы:
- Изменение выборки &ndash; передискретизация данных (*resampling*)
- Взвешивание классов при обучении модели (`class_weight` в моделях `scikit-learn` или вручную)
- Использование других метрик

> ```

## Ресэмплинг данных

В зависимости от характера дисбаланса используются следующие методы:
- увеличение количества наблюдений положительного класса &ndash; дублирование наблюдений или синтезирование новых данных на основе имеющихся по каким-либо правилам
- уменьшение количества наблюдений отрицательного класса &ndash; удаление наблюдений согласно каким-либо правилам
- смешанный подход, комбинирующий вышеуказанные

> Ресэмплинг применяется только к ***обучающей*** выборке. Тестовые данные должны сохранять реальное распределение, иначе оценка модели будет некорректной.

### Методы, увеличивающие количество наблюдений положительного класса

#### RandomOverSampler

Источники:
- [RandomOverSampler @ imblearn](https://imbalanced-learn.org/stable/references/generated/imblearn.over_sampling.RandomOverSampler.html).

Метод генерирует дополнительные наблюдения путем дублирования примеров из миноритарного класса (класса с меньшим 
количеством примеров) случайным образом до выравнивания количества наблюдений миноритарного и мажоритарного классов.

```python
from imblearn.over_sampling import RandomOverSampler
ros = RandomOverSampler(random_state=42)
X_train_resampled, y_train_resampled = ros.fit_resample(X_train, y_train)
```

Основные параметры:
- `sampling_strategy` &ndash; определяет, как именно ресэмплить данные.  
  Может принимать значения:
  + `minority` &ndash; только редкий класс
  + `not minority` &ndash; все, кроме редкого
  + `not majority` &ndash; все, кроме мажоритарного
  + `all` &ndash; все
  + `auto` &ndash; то же самое, что и `not majority`
  + дробное число (желаемое соотношение) &ndash; ***только для бинарного признака***
- `random_state` &ndash; зерно (seed) для инициализации генератора случайных чисел
- `shrinkage` &ndash; позволяет добавить небольшой «шум» к дубликатам, превращая обычный оверсэмплинг в сглаженный (smoothed bootstrap)

Преимущества:
- Простота и скорость: работает быстрее сложных методов вроде `SMOTE`.

Недостатки:
- Риск переобучения: модель может слишком сильно «запомнить» конкретные дублирующиеся примеры.

#### Synthetic Minority Oversampling Technique (SMOTE)

Источники:
- [SMOTE @ imblearn](https://imbalanced-learn.org/stable/references/generated/imblearn.over_sampling.SMOTE.html)
- [Синтетическое генерирование данных (SMOTE) @ Хабр](https://habr.com/ru/companies/otus/articles/782668/)
- [Synthetic Minority Oversampling Technique @ Хабр](https://habr.com/ru/articles/842480/)

Метод находит ближайших соседей миноритарного класса и генерирует образцы, делая линейную интерполяцию между ними. Обычно образец выбирается случайным образом из малочисленного класса. Это делается для того, чтобы обеспечить разнообразие и предотвратить смещение в сторону какой-либо конкретной области пространства признаков. Однако, в некоторых случаях, выбор может быть и целенаправленным, основанным на определенных критериях, например, близости к границе решения. Далее осуществляется поиск его ближайших $k$ соседей. Количество соседей обычно выбирается экспериментально: слишком маленькое $k$ может привести к недостаточному разнообразию, а слишком большое &ndash; к потере специфики малочисленного класса.

Далее на основе выбранного образца и его соседей генерируется синтетический образец путем интерполяции между ними. Новый образец будет располагаться где-то между выбранным образцом и его соседями в пространстве признаков, наследуя их характеристики.

```python
from imblearn.over_sampling import SMOTE
sm = SMOTE(random_state=42)
X_train_resampled, y_train_resampled = sm.fit_resample(X_train, y_train)
```

Основные параметры:
- `sampling_strategy` &ndash; определяет, как именно ресэмплить данные.  
  Может принимать значения:
  + `minority` &ndash; только редкий класс
  + `not minority` &ndash; все, кроме редкого
  + `not majority` &ndash; все, кроме мажоритарного
  + `all` &ndash; все
  + `auto` &ndash; то же самое, что и `not majority`
  + дробное число (желаемое соотношение) &ndash; ***только для бинарного признака***
- `random_state` &ndash; зерно (seed) для инициализации генератора случайных чисел
- `k_neighbors` &ndash; количество ближайших соседей

Преимущества:
- Снижает риск переобучения по сравнению с простым дублированием данных.

Недостатки:
- Может создавать «шум», если классы сильно перемешаны или есть выбросы.
- Плохо работает с категориальными признаками (нужны специальные версии вроде `SMOTE-NC`).
- На данных очень высокой размерности может работать неэффективно.

Вариации `SMOTE`:
- `Borderline-SMOTE`
- `ADASYN`
- `SMOTEN`
- `SMOTENC`
- `KMeansSMOTE`
- `SVMSMOTE`

##### Borderline-SMOTE

Источники:
- [BorderlineSMOTE @ imblearn](https://imbalanced-learn.org/stable/references/generated/imblearn.over_sampling.BorderlineSMOTE.html)

`Borderline-SMOTE` фокусируется на образцах, которые находятся на границе между классами. Идея здесь в том, что эти образцы часто являются наиболее трудными для классификации и, следовательно, наиболее важными для обучения модели.

Сначала идентифицируются граничные образцы малочисленного класса, которые имеют много соседей из многочисленного класса. Затем SMOTE применяется только к этим граничным образцам, увеличивая их количество и тем самым усиливая границу решения вокруг них.

Алгоритм эффективен для моделей, чувствительных к границам классов (например, `SVM`, деревья решений), но может быть менее полезен для мощных ансамблей (`XGBoost`, `CatBoost`), которые часто сами неплохо справляются с дисбалансом через веса классов.

##### Adaptive Synthetic (ADASYN)

Источники:
- [ADASYN @ imblearn](https://imbalanced-learn.org/stable/references/generated/imblearn.over_sampling.ADASYN.html)

`ADASYN` сосредотачивается на образцах, для которых классификация является наиболее сложной, и генерирует больше синтетических данных для этих образцов. Количество синтетических данных для каждого образца определяется на основе количества соседей из другого класса &ndash; чем больше соседей из другого класса, тем больше синтетических данных будет сгенерировано.

Для каждого объекта миноритарного класса алгоритм находит $k$ ближайших соседей. Чем больше среди них представителей мажоритарного класса, тем более «трудным» для обучения считается данный объект. Объектам с высоким уровнем сложности (находящимся в окружении чужого класса) присваиваются большие веса. В результате в этих «сложных» зонах создается больше новых синтетических точек. Новые точки создаются путем линейной интерполяции между выбранным «трудным» объектом и его ближайшими соседями из того же (миноритарного) класса.

##### SMOTEN и SMOTENC

*Synthetic Minority Over-sampling Technique for Nominal* (`SMOTEN`) и *Synthetic Minority Over-sampling Technique for Nominal and Continuous* (`SMOTENC`) предназначены для работы с категориальными данными. При этом `SMOTEN` предназначен исключительно для категориальных данных, а `SMOTENC` применяется в том числе для численных наборов.

Источники:
- [SMOTEN @ imblearn](https://imbalanced-learn.org/stable/references/generated/imblearn.over_sampling.SMOTEN.html)
- [SMOTENC @ imblearn](https://imbalanced-learn.org/stable/references/generated/imblearn.over_sampling.SMOTENC.html)
- [ValueDifferenceMetric @ imblearn](https://imbalanced-learn.org/stable/references/generated/imblearn.metrics.pairwise.ValueDifferenceMetric.html)

Для поиска ближайших соседей вместо привычного евклидова расстояния `SMOTEN` использует специальные метрики, такие как *Value Difference Metric* (`VDM`). `VDM` определяет близость двух категорий на основе их статистической связи с целевым классом.

$$
\delta(x, y) = \sum_{c=1}^{C}{|p(c|x_f) - p(c|y_f)|^k},
$$

где
- $x_f$ и $y_f$ &ndash; два образца признака $f$
- $C$ &ndash; количество классов
- $p(c|x_f)$ &ndash; условная вероятность того, что выходной класс $c$ соответствует значению признака $f$, равному $x$
- $k$ обычно равно 1 или 2.

Расстояние между двумя объектами (векторами признаков) $X$ и $Y$:

$$
\Delta(X,Y) = \sum_{f=1}^{F}\delta(X_f, Y_f)^r,
$$

где
- $F$ &ndash; количество признаков
- $r$ обычно равно 1 или 2.

> По умолчанию в `SMOTEN` для кодирования категориальных признаков используется [OrdinalEncoder](https://scikit-learn.org/stable/modules/generated/sklearn.preprocessing.OrdinalEncoder.html).
>
> `SMOTENC` по умолчанию использует `OneHotEncoder`. `SMOTENC` предполагает наличие хотя бы одного численного признака.

`SMOTENC` для генерации новых образцов числовых признаков использует `SMOTE`. Для категориальных признаков новый образец формируется из наиболее часто встречающегося значения (`mode`) среди тех же $k$ ближайших соседей, которые использовались для числовых признаков. При поиске «ближайших соседей» алгоритм учитывает разницу в категориях, добавляя штраф к евклидову расстоянию, если значения категориальных признаков не совпадают.

##### KMeansSMOTE

`KMeansSMOTE` добавляет этап кластеризации перед применением `SMOTE`:
- сначала метод группирует объекты миноритарного (чаще &ndash; именно только их) в $k$ кластеров с помощью метода кластеризации `KMeans`.
- затем внутри каждого кластера создаются синтетические образцы методом `SMOTE` только между соседями внутри кластера.

Часто распределение количества генерируемых точек между кластерами делается неравномерным (например, пропорционально размеру кластера или наоборот &ndash; больший вес дается более разреженным кластерам), чтобы компенсировать локальные дисбалансы внутри миноритарного класса.

Источники:
- [KMeansSMOTE @ imblearn](https://imbalanced-learn.org/stable/references/generated/imblearn.over_sampling.KMeansSMOTE.html)
- [Борьба с дисбалансом классов. Oversampling @ Хабр](https://habr.com/ru/companies/kozhindev/articles/968714/)

Основные параметры:
- `sampling_strategy` 
- `random_state`
- `k_neighbors`
- `kmeans_estimator` &ndash; инстанс класса алгоритмов `KMeans` (`MiniBatchKMeans` по умолчанию) или количество кластеров
- `cluster_balance_threshold` &ndash; значение порога сбалансированности кластера

Преимущества:
- Снижение шума. Обычный `SMOTE` может генерировать точки между любыми соседями миноритарного класса, даже если они являются выбросами или находятся глубоко в зоне мажоритарного класса. `KMeansSMOTE` ограничивает генерацию только компактными группами (кластерами), что уменьшает риск «размытия» границ между классами.
- Борьба с внутриклассовым дисбалансом. Алгоритм помогает сбалансировать данные не только между классами в целом, но и внутри самого редкого класса, если он состоит из нескольких разрозненных групп.

##### Support Vector Machine Synthetic Minority Over-sampling Technique (SVMSMOTE)

Источники:
- [SVMSMOTE @ imblearn](https://imbalanced-learn.org/stable/references/generated/imblearn.over_sampling.SVMSMOTE.html)

Данный метод отличается от `KMeansSMOTE` тем что, для определения тех экземпляров миноритарного класса, которые следует дополнять, используется метод опорных векторов (`SVM`) вместо `KMeans`.

- Сначала алгоритм обучает классификатор `SVM` на исходных данных.
- Осуществляется поиск опорных векторов миноритарного класса &ndash; тех точек, которые находятся ближе всего к границе с мажоритарным классом.
- Новые синтетические образцы создаются только на основе этих опорных векторов:
  + если большинство соседей точки относятся к её же классу, применяется интерполяция (усиление существующих областей)
  + если много соседей из другого класса, применяется экстраполяция (расширение границ миноритарного класса в сторону мажоритарного)

Преимущества:
- Меньше шума. Не генерирует данные в глубине кластера, где они не несут новой информации для классификатора.
- Точная граница. Усиливает именно те области, где классификатор чаще всего ошибается (границы классов).

### Методы, уменьшающие количество наблюдений отрицательного класса

#### RandomUnderSampler

Источники:
- [RandomUnderSampler @ imblearn](https://imbalanced-learn.org/stable/references/generated/imblearn.under_sampling.RandomUnderSampler.html)
- [Борьба с дисбалансом классов. Undersampling @ Хабр](https://habr.com/ru/companies/kozhindev/articles/954402/)

Алгоритм выравнивает количество экземпляров в классах путем случайного удаления примеров из мажоритарного (преобладающего) класса, пока их число не сравняется с числом примеров в миноритарном классе.

```python
from imblearn.under_sampling import RandomUnderSampler
rus = RandomUnderSampler(random_state=42)
X_train_resampled, y_train_resampled = rus.fit_resample(X_train, y_train)
```

Основные параметры:
- `sampling_strategy`  
  Может принимать значения:
  + `majority` &ndash; только мажоритарный класс
  + `not minority` &ndash; все, кроме редкого
  + `not majority` &ndash; все, кроме мажоритарного
  + `all` &ndash; все
  + `auto` &ndash; то же самое, что и `not minority`
- `random_state`

Преимущества:
- высокая скорость работы
- простота реализации
- значительное уменьшение объема данных

Недостатки:
- риск потери важной информации, так как примеры удаляются вслепую, что может негативно сказаться на точности модели

#### Cluster Centroids

Источники:
- [ClusterCentroids @ imblearn](https://imbalanced-learn.org/stable/references/generated/imblearn.under_sampling.ClusterCentroids.html)
- [Борьба с дисбалансом классов. Undersampling @ Хабр](https://habr.com/ru/companies/kozhindev/articles/954402/)

При данном методе представители мажоритарного класса объединяются в кластеры, после чего они заменяются на одно значение, которое лежит в центре этого кластера. В результате происходит неполное удаление части данных, сохраняется некое представления об удаленных данных. Используется `KMeans` для кластеризации.

#### NearMiss

Источники:
- [NearMiss @ imblearn](https://imbalanced-learn.org/stable/references/generated/imblearn.under_sampling.NearMiss.html)
- [Борьба с дисбалансом классов. Undersampling @ Хабр](https://habr.com/ru/companies/kozhindev/articles/954402/)

Данный метод основан на определении расстояния между частыми и редкими данными. 3 вариации метода:

- `NearMiss-1`  
  Для каждого образца мажоритарного класса вычисляется среднее расстояние до $N$ ближайших соседей миноритарного класса. Остаются те образцы мажоритарного класса, у которых это среднее расстояние наименьшее (то есть они ближе к миноритарному). Полезно, если необходимо сохранить сэмплы, которые находятся рядом с границей.
- `NearMiss-2`  
  Для каждого сэмпла мажоритарного класса вычисляется среднее расстояние до $N$ самых удалённых примеров миноритарного класса.
Остаются те образцы мажоритарного класса, у которых это среднее расстояние наименьшее (то есть они ближе к миноритарному). Часто дает более «компактный» выбор образцов мажоритарного класса.  
- `NearMiss-3`  
   Для каждого образца миноритарного класса выбирается $M$ ближайших соседей мажоритарного. Далее для каждого образца из этого множества (подмножество образцов мажоритарного класса) выбираются те, расстояние от которых до $N$ ближайших соседей из миноритарного класса наибольшее. Гарантирует покрытие миноритарного класса: для каждого образца миноритарного класса берутся ближайшие соседи из мажоритарного.

Основные параметры:
- `sampling_strategy` 
- `version` &ndash; число из $\{1, 2, 3\}$ &ndash; вариация алгоритма `NearMiss`
- `n_neighbors` &ndash; количество ближайших соседей $N$
- `n_neighbors_ver3` &ndash; количество ближайших соседей $M$ для `NearMiss-3`

Преимущества:
- В отличие от случайного удаления, `NearMiss` сохраняет образцы, которые наиболее важны для определения границы между классами

Недостатки:
- Метод может привести к потере важной информации о структуре мажоритарного класса, если границы сильно пересекаются, или сделать модель слишком чувствительной к шуму

#### Edited Nearest Neighbours (ENN)

Источники:
- [EditedNearestNeighbours @ imblearn](https://imbalanced-learn.org/stable/references/generated/imblearn.under_sampling.EditedNearestNeighbours.html)
- [Борьба с дисбалансом классов. Undersampling @ Хабр](https://habr.com/ru/companies/kozhindev/articles/954402/)

Алгоритм находит $k$ ближайших соседей для каждого образца мажоритарного класса. Если все ближайшие соседи относятся к мажоритарному классу, то он удаляется (режим `all`). Данный метод поддерживает и другой алгоритм &ndash; сэмпл будет удаляться, если большинство его соседей относятся к мажоритарному классу (режим `mode`).

Основные параметры:
- `sampling_strategy`
- `n_neighbors` &ndash; количество ближайших соседей $k$
- `kind_sel` &ndash; стратегия алгоритма $\{all, mode\}$. Значение по умолчанию `all`.

Вариации:
- Repeated Edited Nearest Neighbours (RENN)
- All KNN

##### Repeated Edited Nearest Neighbours (RENN)

Источники:
- [RepeatedEditedNearestNeighbours @ imblearn](https://imbalanced-learn.org/stable/references/generated/imblearn.under_sampling.RepeatedEditedNearestNeighbours.html)
- [Борьба с дисбалансом классов. Undersampling @ Хабр](https://habr.com/ru/companies/kozhindev/articles/954402/)

Отличается от метода `ENN` многократным повторением, пока в очередном проходе ничего не удаляется, либо до заранее заданного числа итераций.

Основные параметры:
- `sampling_strategy`
- `n_neighbors`
- `kind_sel`
- `max_iter` &ndash; максимальное количество итераций

##### All KNN

Источники:
- [AllKNN @ imblearn](https://imbalanced-learn.org/stable/references/generated/imblearn.under_sampling.AllKNN.html)
- [Борьба с дисбалансом классов. Undersampling @ Хабр](https://habr.com/ru/companies/kozhindev/articles/954402/)

Если в `ENN` используется фиксированное число соседей (например, $k=3$), то в `AllKNN` выполняется серия шагов, где $k$ последовательно увеличивается от 1 до заданного максимального значения.

#### TomekLinks

Источники:
- [TomekLinks @ imblearn](https://imbalanced-learn.org/stable/references/generated/imblearn.under_sampling.TomekLinks.html)
- [Борьба с дисбалансом классов. Undersampling @ Хабр](https://habr.com/ru/companies/kozhindev/articles/954402/)

Суть данного метода заключается в следующем: если два объекта из разных классов находятся близко, то они находятся в «зоне неопределенности». Данный метод находит такие пары и удаляет представителя мажоритарного класса. Удаляться могут в том числе и представители миноритарного класса, в таком случае данный метод используется для очистки от похожих записей.

Основные параметры:
- `sampling_strategy`

`TomekLinks` часто используют не самостоятельно, а в связке с алгоритмом `SMOTE` при оверсэмплинге:
- `SMOTE` увеличивает количество данных, но часто создает «зашумленные» точки, заходящие на территорию мажоритарного класса
- `TomekLinks` затем «подчищает» эти наложения, удаляя сомнительные пограничные пары

#### Condensed Nearest Neighbour (CNN)

Источники:
- [CondensedNearestNeighbour @ imblearn](https://imbalanced-learn.org/stable/references/generated/imblearn.under_sampling.CondensedNearestNeighbour.html)
- [Борьба с дисбалансом классов. Undersampling @ Хабр](https://habr.com/ru/companies/kozhindev/articles/954402/)

Данный метод заменяет данные на сжатое (condensed) подмножество.

Сжатое подмножество &ndash; это небольшая часть выборки, состоящая из наиболее информативных примеров. Алгоритм `CNN` поддерживает несколько стратегий определения такого подмножества (sampling_strategy). Так при стратегии `majority` все представители редкого класса будут перенесены в сжатое подмножество (*Store*). Остальные примеры идут в несжатое подмножество (*Grab Bag*). Затем поочерёдно берут каждый пример из несжатого и подают вместе со сжатым подмножеством на классификатор 1-NN (классификатор ближайшего соседа). Если пример правильно классифицируется, его пропускают как не несущий никакой дополнительной информации, если классификатор ошибается &ndash; переносят в сжатое. Процесс продолжается до тех пор, пока данные не перестанут добавляться в сжатое множество. В результате остаются только информативные примеры, устранение дисбаланса это только побочный и не обязательный эффект.

Данный метод поддерживает несколько механизмов разделения на подмножества:
- `majority` &ndash; уменьшает только мажоритарный класс (самый многочисленный)
- `not minority` &ndash; уменьшает все классы, кроме самого маленького (при наличии более 2-х классов данных)
- `all` &ndash; уменьшает все классы (в сжатое подмножество добавляются по одному представителю каждого класса, выбранные случайным образом)
- `auto` &ndash; эквивалент 'not minority'
- конкретное количество (или долю) экземпляров для каждого класса в виде словаря
- список конкретных классов, которые нужно уменьшить

Основные параметры:
- `sampling_strategy`
- `random_state`
- `n_neighbors`
- `n_seeds_S` &ndash; количество сэмплов для выбора в подмножество *Store*

Преимущества:
- Значительно уменьшает объем памяти и ускоряет предсказания

Недостатки:
- Алгоритм очень чувствителен к шуму. Выбросы (ошибочные точки) почти всегда попадают в *Store*, так как их сложно правильно классифицировать, что может «загрязнить» обучающую выборку.

#### Neighbourhood Cleaning Rule (NCR)

Источники:
- [NeighbourhoodCleaningRule @ imblearn](https://imbalanced-learn.org/stable/references/generated/imblearn.under_sampling.NeighbourhoodCleaningRule.html)
- [Борьба с дисбалансом классов. Undersampling @ Хабр](https://habr.com/ru/companies/kozhindev/articles/954402/)

Представляет собой попытку объединить метод `ENN` и `CNN`: сначала проводится удаление части данных с помощью `ENN`, оставшиеся данные очищаются с помощью `CNN`. В отличие от *Condensed Nearest Neighbour*, `NCR` больше ориентирован на очистку, чем на радикальное сжатие датасета.

Основные параметры:
- `sampling_strategy`
- `edited_nearest_neighbours` &ndash; инстанс `ENN` или `None` (`ENN` с параметрами `kind_sel="mode"` и `n_neighbors=n_neighbors`)
- `n_neighbors`
- `threshold_cleaning` &ndash; значение порога $\Theta$ (доля сэмплов): $C_i > C \times \Theta$, который определяет, необходимо ли применять `CNN` к классу после применения `ENN`

Преимущества:
- `NCR` выполняет простые локальные очистки и сильно сокращает набор перед тем, как запускать более тяжелые операции, тогда как классический `CNN` &ndash; это итеративный, часто квадратичный по числу расстояний процесс, который многократно сравнивает пары объектов.

#### One Sided Selection (OSS)

Источники:
- [OneSidedSelection @ imblearn](https://imbalanced-learn.org/stable/references/generated/imblearn.under_sampling.OneSidedSelection.html)
- [Борьба с дисбалансом классов. Undersampling @ Хабр](https://habr.com/ru/companies/kozhindev/articles/954402/)

OSS комбинирует две идеи:
- Удаление легко классифицируемых примеров мажоритарного класса. Если объект частого класса находится в своей группе (т.е. его ближайшие соседи так же принадлежат частому классу), то он не несет новой информации и может быть удален.
- Сохранение граничных и сложных объектов. Объекты мажоритарного класса, которые находятся рядом с миноритарным классом, сохраняются &ndash; они помогают алгоритму лучше провести границу между классами.

Алгоритм использует `CNN` для удаления ненужных образцов, а затем применяет `Tomek Links` для очистки границ (удаляет пары «перепутанных» соседей разных классов, когда они мешают корректной классификации).

Основные параметры:
- `sampling_strategy`
- `random_state`
- `n_neighbors`
- `n_seeds_S`

Преимущества:
- Позволяет значительно сократить объем данных мажоритарного класса без потери ключевой информации о границах разделения классов. В отличие от случайного удаления (Random Undersampling), OSS действует направленно, сохраняя «сложные» для классификации примеры.

Недостатки:
- Метод может быть вычислительно затратным на очень больших датасетах из-за использования алгоритма ближайших соседей. Также он лучше всего подходит для умеренно несбалансированных данных и может быть менее эффективен при экстремальном дисбалансе без сочетания с другими методами.

#### Instance Hardness Threshold (IHT)

Источники:
- [InstanceHardnessThreshold @ imblearn](https://imbalanced-learn.org/stable/references/generated/imblearn.under_sampling.InstanceHardnessThreshold.html)
- [Борьба с дисбалансом классов. Undersampling @ Хабр](https://habr.com/ru/companies/kozhindev/articles/954402/)

Данный метод оценивает уверенность классификатора в каждом объекте через предсказанную вероятность его истинного класса, полученную с помощью кросс-валидации. Затем он удаляет из целевых классов те объекты, у которых самая низкая вероятность принадлежности к своему классу (т.е. самые «трудные» для классификатора), чтобы получить нужный баланс.

Основные параметры:
- `estimator` &ndash; модель, которая будет оценивать «трудность» примеров. Можно использовать логистическую регрессию, SVM или деревья решений. Модель должна иметь реализацию метода `predict_proba`.
- `sampling_strategy`  
  Может принимать значения:
  + `majority` &ndash; только мажоритарный класс
  + `not minority` &ndash; все, кроме редкого
  + `not majority` &ndash; все, кроме мажоритарного
  + `all` &ndash; все
  + `auto` &ndash; то же самое, что и `not minority`
  + вещественное число $\alpha_{us} = \frac{N_m}{N_{rM}}$, $N_m$ &ndash; количество сэмплов миноритарного класса, $N_{rM}$ &ndash; количество сэмплов мажоритарного класса после ресэмплинга
- `random_state`
- `cv` &ndash; количество фолдов для кросс-валидации

Преимущества:
- `IHT` помогает очистить границу между классами, удаляя «спорные» примеры, что часто повышает точность модели на редких событиях (например, при поиске мошенничества или диагностике редких болезней).

### Комбинация методов ресэмплинга

#### Synthetic Minority Oversampling Technique + Edited Nearest Neighbors (SMOTEENN)

Источники:
- [SMOTEENN @ imblearn](https://imbalanced-learn.org/stable/references/generated/imblearn.combine.SMOTEENN.html)

Алгоритм выполняет два последовательных этапа:
- `SMOTE` создает новые синтетические примеры для миноритарного класса путем линейной интерполяции между существующими точками и их ближайшими соседями.
- `ENN` очищает полученный набор данных. Алгоритм проверяет каждого «соседа» точки: если класс объекта отличается от класса большинства его ближайших соседей, этот объект удаляется.

Обычный `SMOTE` часто создает «шумные» данные, генерируя синтетические точки в областях, где классы сильно пересекаются. `SMOTEENN` решает эту проблему: 
- Удаляет шум: `ENN` убирает выбросы и примеры, которые оказались слишком глубоко на территории чужого класса
- Улучшает границы: за счет удаления пограничных и сомнительных точек границы между классами становятся более четкими, что упрощает работу классификатора.

#### SMOTETomek

Источники:
- [SMOTETomek @ imblearn](https://imbalanced-learn.org/stable/references/generated/imblearn.combine.SMOTETomek.html)

Алгоритм выполняет два последовательных этапа:
- `SMOTE` создает новые синтетические примеры для миноритарного класса путем линейной интерполяции между существующими точками и их ближайшими соседями.
- `Tomek Links` удаляет пары близко расположенных примеров разных классов, которые находятся на границе разделения.
